这是一个非常经典的 ROS2 进阶场景。由于服务（Service）的回调函数在默认情况下是阻塞的，要在服务回调中实现“持续平滑移动”并“时刻监听自身位置”，我们必须打破单线程的限制。

在这里，我们将使用 **ament_cmake** 构建系统（完全抛弃 `setup.py`），利用你提到的 `install(PROGRAMS ... DESTINATION lib/${PROJECT_NAME})` 方法来赋予 Python 脚本可执行权限并完成安装。同时，我们会引入 **多线程执行器 (MultiThreadedExecutor)** 和 **可重入回调组 (ReentrantCallbackGroup)** 来防止位置监听和服务阻塞产生死锁。

以下是完整的图文与代码实战步骤。

---

### 第一步：创建基于 ament_cmake 的包

打开终端，进入你的工作空间（例如 `~/ros2_ws/src`），创建一个 `ament_cmake` 类型的包：

```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_cmake turtle_nav_pkg
cd turtle_nav_pkg
```

### 第二步：定义自定义服务接口

1. 在包根目录下创建 `srv` 文件夹并新建接口文件：
   ```bash
   mkdir srv
   touch srv/MoveTurtle.srv
   ```
2. 编辑 `srv/MoveTurtle.srv`，写入以下内容（请求为目标坐标，响应为是否成功）：
   ```text
   float64 x
   float64 y
   ---
   bool success
   ```

### 第三步：配置 package.xml

为了让自定义接口正常编译，并包含我们将使用的 Python 依赖，打开 `package.xml`，在 `<buildtool_depend>ament_cmake</buildtool_depend>` 后面补充以下依赖：
```xml
  <depend>rclpy</depend>
  <depend>geometry_msgs</depend>
  <depend>turtlesim</depend>
  
  <!-- 接口生成所需依赖 -->
  <build_depend>rosidl_default_generators</build_depend>
  <exec_depend>rosidl_default_runtime</exec_depend>
  <member_of_group>rosidl_interface_packages</member_of_group>
```

### 第四步：配置 CMakeLists.txt (核心！)

在这个文件中，我们将配置接口生成，并使用你指定的 `install(PROGRAMS ...)` 指令来打包 Python 节点，**不使用任何 setup.py**。

修改 `CMakeLists.txt`：
```cmake
cmake_minimum_required(VERSION 3.8)
project(turtle_nav_pkg)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# 1. 查找依赖
find_package(ament_cmake REQUIRED)
find_package(rclpy REQUIRED)
find_package(geometry_msgs REQUIRED)
find_package(turtlesim REQUIRED)
find_package(rosidl_default_generators REQUIRED)

# 2. 生成自定义服务接口
rosidl_generate_interfaces(${PROJECT_NAME}
  "srv/MoveTurtle.srv"
)

# 3. 为 Python 脚本配置安装与权限
# 这会将 src 下的 Python 脚本安装到 ROS2 的 lib 目录下，使其可通过 ros2 run 调用
install(PROGRAMS
  src/turtle_server.py
  src/turtle_client.py
  DESTINATION lib/${PROJECT_NAME}
)

ament_package()
```

### 第五步：编写服务端和客户端 (Python)

由于我们要让海龟平滑移动，我们将使用基础的 P（比例）控制器：
线速度计算公式为 $v = K_p \cdot \text{distance}$ 
角速度计算公式为 $\omega = K_\theta \cdot \text{angle\_error}$

#### 1. 服务端实现
新建 `src/turtle_server.py`。**注意：为防止服务在 `while` 循环中阻塞导致无法接收 `/turtle1/pose` 话题数据，这里使用了多线程执行器。**
```python
#!/usr/bin/env python3
import math
import time
import rclpy
from rclpy.node import Node
from rclpy.callback_groups import ReentrantCallbackGroup
from rclpy.executors import MultiThreadedExecutor
from geometry_msgs.msg import Twist
from turtlesim.msg import Pose
from turtle_nav_pkg.srv import MoveTurtle # 引入自定义服务

class TurtleMoveServer(Node):
    def __init__(self):
        super().__init__('turtle_move_server')
        # 创建可重入的回调组，允许回调函数并发执行
        self.cb_group = ReentrantCallbackGroup()

        # 发布速度指令
        self.cmd_pub = self.create_publisher(Twist, '/turtle1/cmd_vel', 10)
        
        # 订阅海龟当前位置，加入多线程回调组
        self.pose_sub = self.create_subscription(
            Pose, '/turtle1/pose', self.pose_callback, 10, callback_group=self.cb_group)

        # 创建服务端，加入多线程回调组
        self.srv = self.create_service(
            MoveTurtle, 'move_turtle', self.execute_callback, callback_group=self.cb_group)

        self.current_pose = None
        self.get_logger().info('海龟导航服务端已启动，等待目标坐标...')

    def pose_callback(self, msg):
        """时刻更新自身坐标"""
        self.current_pose = msg

    def execute_callback(self, request, response):
        """处理客户端发送的坐标请求"""
        self.get_logger().info(f'---> 收到新任务: 前往坐标 x={request.x:.2f}, y={request.y:.2f}')

        while self.current_pose is None:
            self.get_logger().warn('尚未获取到海龟初始位置，等待中...')
            time.sleep(0.5)

        twist = Twist()

        # 只要节点存活，且未到达目标点就循环执行
        while rclpy.ok():
            dx = request.x - self.current_pose.x
            dy = request.y - self.current_pose.y
            distance = math.sqrt(dx**2 + dy**2)

            # 距离小于 0.1 视为到达目标
            if distance < 0.1:
                twist.linear.x = 0.0
                twist.angular.z = 0.0
                self.cmd_pub.publish(twist)
                self.get_logger().info('\n==== 任务完成：海龟已抵达目标点！ ====\n')
                response.success = True
                return response

            # 计算 P 控制量
            angle_to_goal = math.atan2(dy, dx)
            angle_error = angle_to_goal - self.current_pose.theta

            # 规范化角度误差到 [-pi, pi]
            while angle_error > math.pi: angle_error -= 2 * math.pi
            while angle_error < -math.pi: angle_error += 2 * math.pi

            # 设置速度并限幅，保证平滑
            twist.linear.x = min(1.5 * distance, 2.0)  # 最大线速度 2.0
            twist.angular.z = 4.0 * angle_error        # 转向比例系数

            self.cmd_pub.publish(twist)
            
            # 时刻输出当前位置
            self.get_logger().info(
                f'当前坐标: ({self.current_pose.x:.2f}, {self.current_pose.y:.2f}) | 距终点: {distance:.2f}', 
                throttle_duration_sec=0.5 # 每0.5秒打印一次防刷屏
            )
            time.sleep(0.05)

        response.success = False
        return response

def main(args=None):
    rclpy.init(args=args)
    node = TurtleMoveServer()
    # 必须使用多线程执行器，否则 while 循环会阻塞 position 的接收
    executor = MultiThreadedExecutor()
    executor.add_node(node)
    
    try:
        executor.spin()
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

#### 2. 客户端实现
新建 `src/turtle_client.py`：
```python
#!/usr/bin/env python3
import sys
import rclpy
from rclpy.node import Node
from turtle_nav_pkg.srv import MoveTurtle

class TurtleMoveClient(Node):
    def __init__(self):
        super().__init__('turtle_move_client')
        self.cli = self.create_client(MoveTurtle, 'move_turtle')
        
        # 等待服务端上线
        while not self.cli.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('等待服务端上线...')
            
        self.req = MoveTurtle.Request()

    def send_request(self, x, y):
        self.req.x = float(x)
        self.req.y = float(y)
        # 异步调用服务
        self.future = self.cli.call_async(self.req)
        # 阻塞等待结果
        rclpy.spin_until_future_complete(self, self.future)
        return self.future.result()

def main(args=None):
    rclpy.init(args=args)

    if len(sys.argv) != 3:
        print("参数错误！\n正确用法: ros2 run turtle_nav_pkg turtle_client.py <x> <y>")
        sys.exit(1)

    client = TurtleMoveClient()
    target_x = sys.argv[1]
    target_y = sys.argv[2]
    
    client.get_logger().info(f'请求海龟移动至坐标: ({target_x}, {target_y})')

    response = client.send_request(target_x, target_y)
    
    if response.success:
        client.get_logger().info('服务端返回：任务顺利完成！')
    else:
        client.get_logger().error('服务端返回：任务失败。')

    client.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 第七步：效果测试

你需要打开三个终端窗口（记得每个终端都要 `source install/setup.bash`）。

**终端 1：启动海龟模拟器**
```bash
ros2 run turtlesim turtlesim_node
```

**终端 2：启动服务端**
```bash
ros2 run turtle_nav_pkg turtle_server.py
```
*(此时你会看到打印：`海龟导航服务端已启动，等待目标坐标...`)*

**终端 3：启动客户端发送坐标** (比如前往坐标 8.0, 8.0)
```bash
ros2 run turtle_nav_pkg turtle_client.py 8.0 8.0
```

当客户端发出请求后：
* 海龟会转身并平滑移动向 `(8.0, 8.0)`。
* 服务端终端会**时刻打印**海龟当前位置与剩余距离。
* 当到达终点后，服务端和客户端都会同时宣布“任务完成”。

### 核心原理解析
由于采用了 `ament_cmake`，传统的 `ament_python` 中的 `setup.py` 被舍弃。`install(PROGRAMS ... DESTINATION lib/${PROJECT_NAME})` 这行 CMake 代码直接将脚本拷贝到了 ROS2 的执行搜索路径下（与 C++ 编译生成的二进制可执行文件同理），配合文件头的 `#!/usr/bin/env python3` 解释器声明，ROS2 就可以完美调度纯 Python 代码，这也是 C++/Python 混合编程包最标准的做法。
