在终端中查看话题的结构并发布指令，是 ROS2 调试中最常用的基本功。

这里需要先做一个小小的概念纠正：在 ROS2 中，节点配置的变量叫**参数 (Parameter)**，而话题中传递的数据结构叫做**消息 (Message)**。你所说的“查看话题的参数”，实际上是**查看该话题的消息接口结构**。

以你提到的**“广播速度”**为例，通常速度话题是 `/cmd_vel`。下面是完整的实操步骤：


在 ros2 topic list -t 命令中，-t 是 --show-types 的缩写（这里的 "t" 代表 types，即“类型”）。

它的作用是在列出当前系统中所有活跃的话题时，同时显示每个话题所绑定的消息类型。



### 第一步：找出话题及其消息类型

如果你还不确定话题的具体名称和类型，可以先列出当前活跃的话题并查看信息。

1. **列出所有话题：**
   ```bash
   ros2 topic list
   ```
   *(假设你看到了 `/cmd_vel`)*

2. **查看该话题的类型：**
   ```bash
   ros2 topic info /cmd_vel
   ```
   **输出示例：**
   ```text
   Type: geometry_msgs/msg/Twist
   Publisher count: 1
   Subscription count: 1


   ```
表示一个订阅一个发布

   *这里我们得到了关键信息：它的消息类型是 `geometry_msgs/msg/Twist`。*

---

### 第二步：查看消息的数据结构（字段）

知道了消息类型后，我们需要知道里面包含哪些“参数”（字段），才能按照格式发送指令。

* **查看接口结构：**
  ```bash
  ros2 interface show geometry_msgs/msg/Twist
  ```
  **输出示例：**
  ```text
  # This expresses velocity in free space broken into its linear and angular parts.
  Vector3  linear
          float64 x
          float64 y
          float64 z
  Vector3  angular
          float64 x
          float64 y
          float64 z
  ```
  *现在你清楚了，要控制速度，需要给 `linear.x` (线速度) 和 `angular.z` (角速度) 等字段赋值。*

---

### 第三步：在终端发布速度指令

使用 `ros2 topic pub` 命令向话题发布数据。数据格式必须符合你在第二步看到的结构，通常使用 **YAML 语法**。

#### 场景 A：持续以特定频率广播（控制真实机器人或仿真器常用）
例如，以 10Hz 的频率发布让机器人以 `0.5 m/s` 前进，并以 `1.0 rad/s` 旋转的指令：
```bash
ros2 topic pub -r 10 /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.5, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 1.0}}"
```
* **`-r 10`**：表示以 10Hz 的频率持续发布（Rate）。
* **`/cmd_vel`**：话题名称。
* **`geometry_msgs/msg/Twist`**：消息类型。
* **`"{...}"`**：YAML 格式的数据。注意字段嵌套的大括号和冒号后面的空格。

#### 场景 B：只发布一次（测试单次触发常用）
如果你只想发送一次指令，可以使用 `--once` 标志（其实如果不加 `-r` 默认也是发送一次）：
```bash
ros2 topic pub --once /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
```
*(这个指令常用于让正在移动的机器人**紧急停止**。)*

> **💡 避坑提示：YAML 格式**
> 在终端中书写 YAML 字符串时，冒号后面**必须有一个空格**（例如 `x: 0.5` 正确，`x:0.5` 报错）。最外层建议用双引号 `""` 包裹起来，防止终端解析出错。
