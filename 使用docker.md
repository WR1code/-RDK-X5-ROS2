掌握 Docker 的常用指令是高效管理容器化应用的基础。为了方便理解和记忆，我们可以将这些指令按照它们操作的对象（**系统、镜像、容器**）进行分类。

以下是 Docker 最核心、最常用的指令详解：

### 一、 基础与系统信息指令

这些指令用于查看 Docker 的运行状态和基本信息。

* **`docker version`**
* **作用：** 查看当前安装的 Docker 客户端和服务端的版本信息。


* **`docker info`**
* **作用：** 显示 Docker 系统的整体信息，包括系统范围内的详细数据（如有多少个容器、多少个镜像、存储驱动、网络配置等）。


* **`docker system prune`**
* **作用：** 清理系统。一键删除所有停止的容器、未被任何容器使用的网络、悬空镜像（dangling images）以及无用的构建缓存。**（注意：操作需谨慎）**



---

### 二、 镜像管理指令 (Image)

镜像是容器的只读模板，相当于面向对象编程中的“类”。

* **`docker search [镜像名]`**
* **作用：** 在 Docker Hub（官方镜像仓库）中搜索指定的镜像。
* *示例：* `docker search nginx`


* **`docker pull [镜像名]:[标签]`**
* **作用：** 从镜像仓库拉取（下载）镜像到本地。如果不指定标签（tag），默认拉取 `latest`（最新版）。
* *示例：* `docker pull ubuntu:20.04`


* **`docker images`** (或 `docker image ls`)
* **作用：** 列出本地已经下载的所有镜像。会显示镜像的 Repository（名称）、Tag（标签）、Image ID、创建时间和大小。


* **`docker rmi [镜像ID或镜像名]`**
* **作用：** 删除本地的一个或多个镜像。如果镜像正在被容器使用，需要先删除容器。使用 `-f` 参数可以强制删除。
* *示例：* `docker rmi nginx:latest`


* **`docker build -t [镜像名]:[标签] [Dockerfile所在目录]`**
* **作用：** 根据指定的 Dockerfile 构建一个新的镜像。
* *示例：* `docker build -t myapp:1.0 .` （`.` 代表当前目录）



---

### 三、 容器生命周期管理指令 (Container)

容器是镜像的运行实例，相当于面向对象编程中的“对象”。

* **`docker run [参数] [镜像名]`**
* **作用：** 创建并启动一个新的容器。这是 Docker 中最复杂、最常用的指令。
* **常用参数：**
* `-d`: 后台运行容器（守护模式），并返回容器 ID。
* `-p [宿主机端口]:[容器内端口]`: 端口映射。例如 `-p 8080:80` 表示将宿主机的 8080 端口映射到容器的 80 端口。
* `-v [宿主机目录]:[容器内目录]`: 目录挂载（数据卷），用于数据持久化。
* `--name [容器名]`: 为容器指定一个自定义名称。
* `-e [环境变量]`: 设置环境变量。
* `-it`: 以交互模式运行容器，并分配一个伪终端（通常用于进入容器命令行）。


* *示例：* `docker run -d -p 8080:80 --name mynginx nginx` (后台启动 nginx，命名为 mynginx，并映射端口)


* **`docker start [容器ID或名字]`**
* **作用：** 启动一个或多个已经被停止的容器。


* **`docker stop [容器ID或名字]`**
* **作用：** 优雅地停止一个正在运行的容器（发送 SIGTERM 信号）。


* **`docker restart [容器ID或名字]`**
* **作用：** 重启容器。


* **`docker rm [容器ID或名字]`**
* **作用：** 删除一个或多个已经停止的容器。
* *进阶：* `docker rm -f [容器ID]` 可以强制删除正在运行的容器。



---

### 四、 容器运维与交互指令

当容器运行起来后，我们需要查看它的状态、日志或进入其内部。

* **`docker ps`**
* **作用：** 列出当前正在运行的容器。
* **常用参数：** `-a` (列出所有容器，包括已经停止的)。


* **`docker exec -it [容器ID或名字] [执行的命令]`**
* **作用：** 在正在运行的容器内部执行命令。这是最常用的**进入容器内部**的方法。
* *示例：* `docker exec -it mynginx /bin/bash` （进入名为 mynginx 的容器，并打开 bash 终端。如果镜像是 alpine 系统，通常用 `/bin/sh`）


* **`docker logs [参数] [容器ID或名字]`**
* **作用：** 获取容器的输出日志。这对于排查服务报错非常关键。
* **常用参数：**
* `-f`: 持续跟踪日志输出（类似 `tail -f`）。
* `--tail [行数]`: 仅显示最后几行日志，例如 `--tail 100`。




* **`docker inspect [容器或镜像ID]`**
* **作用：** 返回 Docker 对象（容器、镜像、网络等）的底层详细信息，以 JSON 格式输出。可以用来查看容器的内部 IP 地址、挂载的数据卷等详细配置。


* **`docker cp [宿主机路径] [容器ID]:[容器内路径]`** (或反向操作)
* **作用：** 在宿主机和容器之间拷贝文件或目录。
* *示例 1（宿主到容器）：* `docker cp index.html mynginx:/usr/share/nginx/html/`
* *示例 2（容器到宿主）：* `docker cp mynginx:/etc/nginx/nginx.conf ./`



### 💡 学习建议

在日常使用中，如果你忘记了某个指令的具体用法，可以直接在命令行输入 `docker [指令] --help`（例如 `docker run --help`），系统会打印出非常详细的参数说明。
