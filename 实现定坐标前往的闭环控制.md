### 第一步：创建工作空间和 CMake 功能包

打开终端，创建一个新的工作空间，并使用 `ament_cmake` 作为构建类型创建功能包。我们同时声明它依赖于 `rclcpp`、`geometry_msgs` 和 `turtlesim`。

```bash
# 1. 创建工作空间并进入 src 目录
mkdir -p ~/turtle_cpp_ws/src
cd ~/turtle_cpp_ws/src

# 2. 创建名为 turtle_cpp_controller 的 C++ 功能包
ros2 pkg create --build-type ament_cmake turtle_cpp_controller --dependencies rclcpp geometry_msgs turtlesim
```

### 第二步：编写 C++ 节点代码 (面向对象封装)

进入功能包的源码目录，创建我们的 C++ 源文件：

```bash
cd ~/turtle_cpp_ws/src/turtle_cpp_controller/src
touch go_to_goal.cpp
```

使用文本编辑器打开 `go_to_goal.cpp`，将以下代码粘贴进去。注意看我是如何将接口暴露在 `public` 中，而将核心控制逻辑、回调函数和状态变量隐藏在 `private` 中的：

```cpp
// ==========================================
// 头文件引入部分 (告诉编译器我们需要用到哪些现成的工具)
// ==========================================
#include <rclcpp/rclcpp.hpp>             // ROS2 C++ 编程的核心库，包含了节点、定时器、日志等所有基础功能
#include <geometry_msgs/msg/twist.hpp>   // ROS2 标准消息类型：Twist（包含线速度和角速度，专门用来控制移动）
#include <turtlesim/msg/pose.hpp>        // turtlesim 专属消息类型：Pose（包含 x, y 坐标和 theta 朝向角度）
#include <cmath>                         // C++ 数学库，提供开根号(sqrt)、乘方(pow)、反正切(atan2)等数学运算
#include <iostream>                      // C++ 标准输入输出流，用于在终端打印文字和接收用户输入

// ==========================================
// 类的定义：创建一个叫做 GoToGoalNode 的 ROS2 节点
// public rclcpp::Node 表示我们的类“继承”了 ROS2 标准节点的所有功能
// ==========================================
class GoToGoalNode : public rclcpp::Node
{
public:
    // 【构造函数】：当这个类被实例化（创建对象）时，最先且只运行一次的代码。
    // 这里接收用户输入的两个参数：目标 X 坐标和目标 Y 坐标
    GoToGoalNode(double target_x, double target_y)
    : Node("go_to_goal_node"),       // 初始化 ROS2 节点的名称，在网络中这个名字是唯一的
      target_x_(target_x),           // 把用户传入的目标X保存到类的私有变量中（带下划线是为了区分局部变量）
      target_y_(target_y),           // 把用户传入的目标Y保存到类的私有变量中
      goal_reached_(false)           // 初始状态下，我们还没有到达目标点，所以设为 false
    {
        // ---------------------------------------------------------
        // 1. 初始化发布者 (Publisher) —— 它的作用是向外发送速度指令
        // ---------------------------------------------------------
        // this->create_publisher<消息类型>("话题名称", 队列长度);
        // /turtle1/cmd_vel 是小乌龟接收速度指令的话题，队列长度 10 表示最多缓存 10 条消息
        publisher_ = this->create_publisher<geometry_msgs::msg::Twist>("/turtle1/cmd_vel", 10);
        
        // ---------------------------------------------------------
        // 2. 初始化订阅者 (Subscriber) —— 它的作用是接收小乌龟当前的实时位置
        // ---------------------------------------------------------
        // 当收到消息时，需要有一个函数来处理它。这就是“回调函数”。
        // std::bind 的作用是把类内部的成员函数 pose_callback 和当前对象(this)绑定起来。
        // std::placeholders::_1 代表留一个空位，等收到了位置消息，就填在这个空位传给回调函数。
        subscriber_ = this->create_subscription<turtlesim::msg::Pose>(
            "/turtle1/pose", 10, std::bind(&GoToGoalNode::pose_callback, this, std::placeholders::_1));
        
        // ---------------------------------------------------------
        // 3. 初始化定时器 (Timer) —— 它的作用是像节拍器一样，每隔固定时间执行一次控制逻辑
        // ---------------------------------------------------------
        // std::chrono::milliseconds(100) 表示 100 毫秒（即 0.1 秒）。
        // 也就是说，我们每秒钟会计算并发送 10 次速度指令给小乌龟（控制频率为 10Hz）。
        timer_ = this->create_wall_timer(
            std::chrono::milliseconds(100), std::bind(&GoToGoalNode::control_loop, this));
            
        // 打印一条绿色的提示信息到终端，RCLCPP_INFO 是 ROS2 专用的打印日志工具
        RCLCPP_INFO(this->get_logger(), "目标已设定为: X=%.2f, Y=%.2f。开始移动...", target_x_, target_y_);
    }

private:
    // ==========================================
    // 核心业务函数区 (私有，只能在类内部调用)
    // ==========================================

    // 【位置回调函数】：每当小乌龟的位置发生变化，并在 /turtle1/pose 话题上广播时，这个函数就会被自动触发
    void pose_callback(const turtlesim::msg::Pose::SharedPtr msg)
    {
        // msg 是一个指向接收到的位置消息的智能指针
        // 我们把指针指向的实际数据 (*msg) 赋值给类的内部变量 current_pose_，这样我们随时都知道乌龟在哪了
        current_pose_ = *msg;
    }

    // 【核心控制循环】：由定时器触发，每 0.1 秒运行一次。这里用的是经典的 P-Controller (比例控制器)
    void control_loop()
    {
        // 如果已经到达目标，就什么也不做，直接结束函数 (return)
        if (goal_reached_) {
            return;
        }

        // ---------------------------------------------------------
        // 步骤 A：计算当前位置到目标位置的直线距离 (勾股定理)
        // ---------------------------------------------------------
        // distance = √((X目标 - X当前)² + (Y目标 - Y当前)²)
        double distance = std::sqrt(std::pow(target_x_ - current_pose_.x, 2) + 
                                    std::pow(target_y_ - current_pose_.y, 2));

        // 准备一个空的速度消息对象，里面包含我们要发送的线速度和角速度 (初始都是 0)
        auto twist_msg = geometry_msgs::msg::Twist();

        // 我们设定一个“容忍度”。如果距离小于 0.1 单位，我们就认为已经非常接近，算作到达目标了
        if (distance >= 0.1) {
            
            // ---------------------------------------------------------
            // 步骤 B：计算线速度 (往前走多快) —— 距离越远开越快，越近开越慢
            // ---------------------------------------------------------
            // 线速度 = 比例系数 (1.0) * 距离误差。这就是 P(比例) 控制的精髓，能实现平滑减速，防止冲过头。
            twist_msg.linear.x = 1.0 * distance;

            // ---------------------------------------------------------
            // 步骤 C：计算角速度 (怎么转向) —— 应该转多少度才能对准目标？
            // ---------------------------------------------------------
            // 1. 算出目标点相对当前点的绝对方位角。std::atan2(delta_Y, delta_X) 会返回一个 [-pi, pi] 之间的角度。
            double angle_to_goal = std::atan2(target_y_ - current_pose_.y, target_x_ - current_pose_.x);
            
            // 2. 算出角度误差：目标角度 - 小乌龟当前正在面朝的角度 (theta)
            double angle_error = angle_to_goal - current_pose_.theta;

            // 3. 规范化角度误差：这一步非常关键！
            // 角度是圆周循环的，359度和1度其实只差了2度，不是358度。
            // 这两个 while 循环确保计算出来的转动角度永远在 -180度 到 +180度 (即 -pi 到 +pi) 之间。
            // 这样小乌龟总是会选择最短的旋转方向，而不会原地傻傻地转大圈。
            while (angle_error > M_PI) angle_error -= 2.0 * M_PI;
            while (angle_error < -M_PI) angle_error += 2.0 * M_PI;

            // 4. 计算并赋予角速度：角速度 = 比例系数 (4.0) * 角度误差。
            // 系数给大一点 (4.0)，是为了让乌龟优先把车头对准目标，再往前跑。
            twist_msg.angular.z = 4.0 * angle_error;

            // ---------------------------------------------------------
            // 步骤 D：发布指令并打印状态
            // ---------------------------------------------------------
            // 把算好的速度指令发送出去，小乌龟接收到就会动起来
            publisher_->publish(twist_msg);

            // 打印当前的坐标和实时的计算速度，方便我们在终端里监控
            RCLCPP_INFO(this->get_logger(), 
                        "当前坐标: X=%.2f, Y=%.2f | 当前速度: 线速度=%.2f, 角速度=%.2f",
                        current_pose_.x, current_pose_.y, twist_msg.linear.x, twist_msg.angular.z);
                        
        } else {
            // =========================================================
            // 当距离 < 0.1，说明到达终点了，执行刹车逻辑
            // =========================================================
            twist_msg.linear.x = 0.0;   // 线速度归零
            twist_msg.angular.z = 0.0;  // 角速度归零
            publisher_->publish(twist_msg); // 发布停止指令
            
            RCLCPP_INFO(this->get_logger(), "🎉 已成功到达目标位置！");
            goal_reached_ = true; // 标记状态为已到达
            
            // 任务完成，呼叫 ROS2 系统关闭当前节点程序
            rclcpp::shutdown();
        }
    }

    // ==========================================
    // 类的成员变量区 (保存数据的“仓库”)
    // 私有成员变量：隐藏内部状态，保护数据不被外部随意篡改
    // ==========================================
    double target_x_;                       // 目标 X 坐标
    double target_y_;                       // 目标 Y 坐标
    bool goal_reached_;                     // 是否到达的标志位
    turtlesim::msg::Pose current_pose_;     // 存储最新的小乌龟实时位置信息

    // ROS2 通信对象：这里用 SharedPtr (共享智能指针) 来管理内存，避免内存泄漏，这是 ROS2 标准做法
    rclcpp::Publisher<geometry_msgs::msg::Twist>::SharedPtr publisher_;  // 速度发布者
    rclcpp::Subscription<turtlesim::msg::Pose>::SharedPtr subscriber_;   // 位置订阅者
    rclcpp::TimerBase::SharedPtr timer_;                                 // 控制循环定时器
};

// ==========================================
// 主函数 (C++ 程序的统一入口)
// ==========================================
int main(int argc, char **argv)
{
    // 获取用户输入，这个时候我们还没有初始化 ROS2
    std::cout << "=== 小乌龟 C++ 平滑导航控制器 ===" << std::endl;
    double target_x, target_y;
    
    // 提示用户输入 X 坐标并检查输入是否合法
    std::cout << "请输入目标 X 坐标 (通常 0.0 ~ 11.0): ";
    if (!(std::cin >> target_x)) {
        std::cerr << "输入错误！请输入数字。" << std::endl;
        return 1; // 非正常退出，返回非 0 错误码
    }
    
    // 提示用户输入 Y 坐标并检查输入是否合法
    std::cout << "请输入目标 Y 坐标 (通常 0.0 ~ 11.0): ";
    if (!(std::cin >> target_y)) {
        std::cerr << "输入错误！请输入数字。" << std::endl;
        return 1;
    }

    // 初始化 ROS2 环境。argc 和 argv 是命令行参数，ROS2 需要用到它们
    rclcpp::init(argc, argv);
    
    // 实例化我们的节点对象。这里使用 std::make_shared 生成一个指向我们节点的智能指针。
    // 并把刚才用户输入的 target_x 和 target_y 传进构造函数里。
    auto node = std::make_shared<GoToGoalNode>(target_x, target_y);
    
    // rclcpp::spin 的作用是“死循环等待”。
    // 它会让程序暂停在这里，并且不断地在后台处理订阅者的消息回调和定时器的触发。
    // 直到遇到 `rclcpp::shutdown()` 被调用，或者用户在终端按下了 Ctrl+C。
    rclcpp::spin(node);
    
    // 扫尾工作：清理和关闭 ROS2 相关的资源。
    // 如果程序内部（如到达目标后）调用了 shutdown，这里的调用会直接忽略，不会报错。
    rclcpp::shutdown();
    
    return 0; // 程序完美运行并结束
}
```

### 第三步：配置 CMakeLists.txt

返回功能包的根目录，打开 `CMakeLists.txt` 文件，我们需要告诉 CMake 如何编译这个 `.cpp` 文件。

找到 `# find_package(<dependency> REQUIRED)` 的位置，并在文件底部（但在 `ament_package()` 之前）添加编译指令：

```cmake
cmake_minimum_required(VERSION 3.8)
project(turtle_cpp_controller)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# 寻找依赖包 (创建包时已经自动生成)
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(geometry_msgs REQUIRED)
find_package(turtlesim REQUIRED)

# ================= 新增配置部分 =================

# 1. 声明可执行文件，源文件为 src/go_to_goal.cpp
add_executable(go_to_goal src/go_to_goal.cpp)

# 2. 为可执行文件添加依赖
ament_target_dependencies(go_to_goal rclcpp geometry_msgs turtlesim)

# 3. 配置安装路径，以便 ros2 run 能找到它
install(TARGETS
  go_to_goal
  DESTINATION lib/${PROJECT_NAME}
)

# ===============================================

ament_package()
```

### 第四步：编译并运行

1. **编译工作空间：**
   回到工作空间的根目录进行编译。

   ```bash
   cd ~/turtle_cpp_ws
   colcon build --packages-select turtle_cpp_controller
   ```

2. **刷新环境变量：**
   每次编译后或打开新终端时，都需要刷新环境。
   ```bash
   source install/setup.bash
   ```

3. **运行测试：**
   首先，在一个新的终端启动 turtlesim 节点：
   ```bash
   ros2 run turtlesim turtlesim_node
   ```

   然后，在刷新了环境的终端中运行你写的 C++ 控制器：
   ```bash
   ros2 run turtle_cpp_controller go_to_goal
   ```

程序启动后，会在终端要求你输入坐标，输入完毕后，小乌龟将平滑移动，并且终端会持续打印当前的坐标和速度信息。这个版本完全遵循了 C++ OOP 的规范，保证了数据的安全性和逻辑的模块化。
