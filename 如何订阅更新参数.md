在ROS 2中，节点可以在运行时动态更改参数。为了监听这些变化，无论是使用 Python (rclpy) 还是 C++ (rclcpp)，你都可以通过注册**参数更新回调函数（Parameter Event Callback）**来实现。

当外部（例如通过命令行 `ros2 param set` 或其他节点）尝试修改参数时，这个回调函数会被触发。**非常重要的一点是：你的回调函数需要返回一个结果，告诉ROS 2是否允许这次参数修改。**

以下是分别使用 Python 和 C++ 的实现方法：

### 1. 使用 Python (rclpy) 订阅参数更新

在 Python 中，你可以使用 `add_on_set_parameters_callback` 方法。

```python
import rclpy
from rclpy.node import Node
from rcl_interfaces.msg import SetParametersResult
from rclpy.parameter import Parameter

class ParamListenerNode(Node):
    def __init__(self):
        super().__init__('param_listener_node')
        
        # 1. 声明一个参数并赋予默认值
        self.declare_parameter('my_string_param', 'default_value')
        self.declare_parameter('my_int_param', 10)

        # 2. 注册参数更新的回调函数
        self.add_on_set_parameters_callback(self.parameter_callback)
        
        self.get_logger().info('节点已启动，正在监听参数更新...')

    def parameter_callback(self, params):
        """
        当有参数被修改时，此回调会被触发。
        :param params: 包含被修改参数列表的 Parameter 对象
        :return: SetParametersResult 对象
        """
        for param in params:
            if param.name == 'my_string_param':
                if param.type_ == Parameter.Type.STRING:
                    self.get_logger().info(f'收到 my_string_param 更新: {param.value}')
                else:
                    self.get_logger().warn('my_string_param 类型错误！拒绝更新。')
                    return SetParametersResult(successful=False, reason="类型必须是字符串")
                    
            elif param.name == 'my_int_param':
                if param.type_ == Parameter.Type.INTEGER:
                    self.get_logger().info(f'收到 my_int_param 更新: {param.value}')
                else:
                    return SetParametersResult(successful=False, reason="类型必须是整数")

        # 3. 必须返回 successful=True，否则参数修改会被拒绝
        return SetParametersResult(successful=True, reason="参数更新成功")

def main(args=None):
    rclpy.init(args=args)
    node = ParamListenerNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 2. 使用 C++ (rclcpp) 订阅参数更新

在 C++ 中，你需要保存一个 `OnSetParametersCallbackHandle::SharedPtr` 句柄，否则回调函数在超出作用域后会被销毁，导致无法监听到更新。

```cpp
#include <rclcpp/rclcpp.hpp>
#include <rcl_interfaces/msg/set_parameters_result.hpp>
#include <string>
#include <vector>

class ParamListenerNode : public rclcpp::Node
{
public:
    ParamListenerNode() : Node("param_listener_node")
    {
        // 1. 声明参数
        this->declare_parameter<std::string>("my_string_param", "default_value");
        this->declare_parameter<int>("my_int_param", 10);

        // 2. 绑定并注册回调函数，保存句柄以防被销毁
        callback_handle_ = this->add_on_set_parameters_callback(
            std::bind(&ParamListenerNode::parameters_callback, this, std::placeholders::_1)
        );

        RCLCPP_INFO(this->get_logger(), "节点已启动，正在监听参数更新...");
    }

private:
    // 定义保存回调句柄的成员变量
    rclcpp::node_interfaces::OnSetParametersCallbackHandle::SharedPtr callback_handle_;

    // 3. 回调函数实现
    rcl_interfaces::msg::SetParametersResult parameters_callback(const std::vector<rclcpp::Parameter> &parameters)
    {
        rcl_interfaces::msg::SetParametersResult result;
        result.successful = true;
        result.reason = "参数更新成功";

        for (const auto &param : parameters)
        {
            if (param.get_name() == "my_string_param")
            {
                if (param.get_type() == rclcpp::ParameterType::PARAMETER_STRING) {
                    RCLCPP_INFO(this->get_logger(), "收到 my_string_param 更新: %s", param.as_string().c_str());
                } else {
                    result.successful = false;
                    result.reason = "类型必须是字符串";
                }
            }
            else if (param.get_name() == "my_int_param")
            {
                if (param.get_type() == rclcpp::ParameterType::PARAMETER_INTEGER) {
                    RCLCPP_INFO(this->get_logger(), "收到 my_int_param 更新: %ld", param.as_int());
                } else {
                    result.successful = false;
                    result.reason = "类型必须是整数";
                }
            }
        }

        return result;
    }
};

int main(int argc, char **argv)
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<ParamListenerNode>());
    rclcpp::shutdown();
    return 0;
}
```

**不完全是这样的。** 

在 ROS 2 中，参数**默认确实是动态的**（可以在运行时随时修改），但你完全可以通过代码将特定参数声明为**只读（Read-only）**，也就是你所说的“静态”参数。

一旦将参数设置为只读，它在初始化之后就**绝对不允许被修改**。任何尝试修改它的外部请求（比如通过命令行 `ros2 param set`）都会被 ROS 2 底层直接拒绝，甚至都不会触发你在上一篇代码中写的 `on_set_parameters_callback` 回调函数。

要实现“静态/只读”参数，你需要使用 `ParameterDescriptor`（参数描述符）。以下是如何在代码中实现：

### 1. Python 中声明只读参数

在声明参数时，传入一个 `read_only=True` 的描述符：

```python
from rclpy.node import Node
from rcl_interfaces.msg import ParameterDescriptor

class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')
        
        # 创建一个参数描述符，设为只读
        static_desc = ParameterDescriptor(
            description='这是一个只读的静态参数',
            read_only=True
        )
        
        # 声明参数并传入描述符
        self.declare_parameter('my_static_param', 'fixed_value', static_desc)
```

### 2. C++ 中声明只读参数

在 C++ 中，逻辑完全相同：

```cpp
#include <rclcpp/rclcpp.hpp>
#include <rcl_interfaces/msg/parameter_descriptor.hpp>

class MyNode : public rclcpp::Node
{
public:
    MyNode() : Node("my_node")
    {
        // 创建一个参数描述符，设为只读
        rcl_interfaces::msg::ParameterDescriptor static_desc;
        static_desc.description = "这是一个只读的静态参数";
        static_desc.read_only = true;

        // 声明参数并传入描述符
        this->declare_parameter<std::string>("my_static_param", "fixed_value", static_desc);
    }
};
```

### 总结一下 ROS 2 的参数修改机制：

1. **默认情况（动态）：** 任何节点或命令行都可以尝试修改它。你需要通过我之前提到的 `add_on_set_parameters_callback` 来决定是“接受”还是“拒绝”这次修改。
2. **只读情况（静态）：** 只要你在代码里设置了 `read_only=true`，这个参数的值就彻底“锁死”了。外部一旦尝试修改，命令行会直接报错：`Setting parameter failed: parameter 'my_static_param' cannot be set because it is read-only`。

所以，如果你有某些参数（比如机器人的轴距、最大物理限速、硬件接口名称等）在启动后绝对不能变，**强烈建议**在声明时加上 `read_only=true` 的描述符，这比在回调函数里手动写 `if/else` 去拒绝修改要安全、规范得多。
