开发一个带有语音播报功能的 ROS 2 自动巡航功能包，核心思路是将导航系统（Nav2）**与**文字转语音系统（TTS, Text-to-Speech）结合起来。

为了实现解耦和高复用性，我们通常采用**多节点架构**：一个节点负责控制巡航逻辑并发布语音指令，另一个节点专门负责接收指令并播放语音。

以下是使用 Python 编写该功能包的详细完整指南，适用于 ROS 2 (Foxy / Humble / Iron)。

---

### 第一步：环境准备与依赖安装

1. **基础依赖**：确保你已经安装了 ROS 2 和 Nav2。
2. **语音播报库**：我们将使用 `pyttsx3`，它是一个跨平台的离线 Python TTS 库。
```bash
pip3 install pyttsx3

```


*(在 Ubuntu 上，可能还需要安装音频系统依赖：`sudo apt install espeak alsa-utils`)*

---

### 第二步：创建 ROS 2 功能包

打开终端，进入你的 ROS 2 工作空间（例如 `~/ros2_ws/src`），创建一个基于 Python 的功能包：

```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python auto_cruise_voice --dependencies rclpy std_msgs geometry_msgs nav2_msgs

```

### 第三步：编写核心代码

在 `~/ros2_ws/src/auto_cruise_voice/auto_cruise_voice/` 目录下，创建两个 Python 脚本文件：`voice_node.py` 和 `cruise_node.py`。

#### 1. 语音播报节点 (`voice_node.py`)

该节点订阅 `/voice_cmd` 话题（类型为 `std_msgs/String`），收到字符串后转化为语音播放。

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String
import pyttsx3
import threading

class VoiceNode(Node):
    def __init__(self):
        super().__init__('voice_node')
        # 初始化语音引擎
        self.engine = pyttsx3.init()
        # 设置语速
        self.engine.setProperty('rate', 150) 
        
        # 创建订阅者
        self.subscription = self.create_subscription(
            String,
            'voice_cmd',
            self.listener_callback,
            10)
        self.get_logger().info('语音播报节点已启动，等待指令...')

    def listener_callback(self, msg):
        self.get_logger().info(f'收到语音指令: "{msg.data}"')
        # 使用多线程播放语音，防止阻塞 ROS 2 回调
        threading.Thread(target=self.speak, args=(msg.data,)).start()

    def speak(self, text):
        self.engine.say(text)
        self.engine.runAndWait()

def main(args=None):
    rclpy.init(args=args)
    node = VoiceNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

```

#### 2. 自动巡航控制节点 (`cruise_node.py`)

该节点使用 ROS 2 Action 客户端连接 Nav2 的 `NavigateToPose`，按顺序发布目标点，并在关键节点发布语音话题。

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from std_msgs.msg import String
from geometry_msgs.msg import PoseStamped
from nav2_msgs.action import NavigateToPose
import time

# [新增] 如果实际开发中需要处理图像，通常需要以下库
# from sensor_msgs.msg import Image
# from cv_bridge import CvBridge
# import cv2

class CruiseNode(Node):
    def __init__(self):
        super().__init__('cruise_node')
        
        # 创建语音指令发布者
        self.voice_pub = self.create_publisher(String, 'voice_cmd', 10)
        
        # 创建 Nav2 导航动作客户端
        self.nav_client = ActionClient(self, NavigateToPose, 'navigate_to_pose')
        
        # [修改] 巡航点字典中增加 'task' 字段，用于标记到达后需要执行的操作
        self.waypoints = [
            {'name': 'A点', 'x': 1.0, 'y': 1.0, 'w': 1.0, 'task': 'vision'}, # 到达后执行图像识别
            {'name': 'B点', 'x': 2.0, 'y': -1.0, 'w': 1.0, 'task': 'none'},   # 无特殊任务
            {'name': '原点', 'x': 0.0, 'y': 0.0, 'w': 1.0, 'task': 'none'}
        ]
        
        # 等待动作服务器就绪
        self.get_logger().info('等待 Nav2 动作服务器就绪...')
        self.nav_client.wait_for_server()
        self.get_logger().info('Nav2 已连接，开始巡航！')
        
        # 启动巡航任务
        self.start_cruising()

    def speak(self, text):
        """发布语音播报指令"""
        msg = String()
        msg.data = text
        self.voice_pub.publish(msg)
        self.get_logger().info(f"🎤 语音播报: {text}") # 终端也打印一下方便调试

    def start_cruising(self):
        self.speak("系统启动，准备开始自动巡航")
        time.sleep(2) # 等待语音播报完毕
        
        for point in self.waypoints:
            self.speak(f"正在前往{point['name']}")
            success = self.send_goal(point)
            
            if success:
                self.speak(f"已成功到达{point['name']}")
                
                # [新增] 检查该点位是否有分配特殊任务
                task = point.get('task', 'none')
                if task == 'vision':
                    self.execute_vision_task(point['name'])
                else:
                    # 如果没有特殊任务，默认停留2秒
                    time.sleep(2) 
            else:
                self.speak(f"前往{point['name']}失败，请检查路径")
                break
                
        self.speak("所有巡航任务已完成")

    # [新增] 图像识别任务的具体执行逻辑
    def execute_vision_task(self, location_name):
        """处理图像识别任务"""
        self.speak(f"开启视觉系统，正在对{location_name}进行扫描")
        self.get_logger().info('📸 正在开启相机，准备接收图像并进行识别...')
        
        # --- 这里放置你的图像识别业务逻辑 ---
        # 实际开发中常见的两种处理方式：
        # 1. 话题订阅法：订阅 '/camera/image_raw' 获取最新的一帧图像，传给 OpenCV/YOLO 模型推理。
        # 2. 服务调用法 (推荐)：调用一个独立的 "图像识别 Service"，等待它返回识别结果（如识别出的人名、物体坐标等）。
        
        time.sleep(3) # 这里用 sleep 模拟图像识别所花费的时间
        
        # 假设我们识别到了某个目标
        mock_detected_object = "红色目标" 
        
        self.get_logger().info(f'✅ 图像识别完成，发现: {mock_detected_object}')
        self.speak(f"识别完成，前方发现{mock_detected_object}")
        time.sleep(2) # 等待播报结束

    def send_goal(self, point):
        """发送导航目标点并等待结果"""
        goal_msg = NavigateToPose.Goal()
        goal_msg.pose.header.frame_id = 'map'
        goal_msg.pose.header.stamp = self.get_clock().now().to_msg()
        
        goal_msg.pose.pose.position.x = point['x']
        goal_msg.pose.pose.position.y = point['y']
        goal_msg.pose.pose.orientation.w = point['w']

        self.get_logger().info(f"发送目标点: {point['name']}")
        
        # 异步发送目标
        send_goal_future = self.nav_client.send_goal_async(goal_msg)
        rclpy.spin_until_future_complete(self, send_goal_future)
        
        goal_handle = send_goal_future.result()
        if not goal_handle.accepted:
            self.get_logger().error('目标点被拒绝')
            return False

        # 等待结果
        result_future = goal_handle.get_result_async()
        rclpy.spin_until_future_complete(self, result_future)
        
        result = result_future.result()
        # 状态 4 代表 SUCCEEDED
        return result.status == 4

def main(args=None):
    rclpy.init(args=args)
    node = CruiseNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 第四步：配置编译文件

打开功能包目录下的 `setup.py`，配置程序的入口点（Entry Points），让 ROS 2 能够找到这两个节点。

在 `entry_points` 的 `console_scripts` 列表中添加：

```python
    entry_points={
        'console_scripts': [
            'voice_node = auto_cruise_voice.voice_node:main',
            'cruise_node = auto_cruise_voice.cruise_node:main',
        ],
    },

```

### 第五步：编写 Launch 文件（可选但推荐）

在功能包根目录下创建一个 `launch` 文件夹，并添加一个 `cruise_voice.launch.py` 文件，用于一键启动两个节点：

```python
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        Node(
            package='auto_cruise_voice',
            executable='voice_node',
            name='voice_node',
            output='screen'
        ),
        Node(
            package='auto_cruise_voice',
            executable='cruise_node',
            name='cruise_node',
            output='screen'
        )
    ])

```

*注意：要在 `setup.py` 的 `data_files` 中将 `launch` 文件夹安装进去。*

---

### 架构优势总结

* **异步处理**：语音节点使用了 `threading`，确保长语音播放时不会卡死 ROS 2 的通讯和话题接收。
* **高内聚低耦合**：巡航节点不需要关心 TTS 引擎是如何运作的，只需要发字符串即可；这意味着你随时可以把 `pyttsx3` 替换成百度语音、科大讯飞等更高级的云端 API，而无需修改巡航逻辑。
